# Code Review — 2026-04-09

## Summary

The app is focused and well-scoped. The shared-code strategy via `Core/` is sound. The main concerns are Swift 6 concurrency annotation gaps, a verbatim copy-paste of the timer callback between the two platform targets, and several smaller readability and safety issues documented below.

Issues are grouped by file and ranked at the end by priority.

---

## `Phase.swift`

**`@Observable` missing `@MainActor`**
Every `@Observable` class must be `@MainActor`. `Phase` is missing the annotation.

**Method name violates Swift naming conventions**
`CycleCompleted()` is UpperCamelCase — that is a type name, not a method name. It must be `cycleCompleted()`.

**Self-naming collision**
The class `Phase` has a stored property also named `phase`. Reading `phase.phase` in calling code is confusing. The integer property should be renamed — `step` or `value` are natural choices.

**`cycleCompleted` is a stateful side-effect flag**
Setting `cycleCompleted = false` inside `advance()` and reading it via `cycleCompleted()` is a subtle gotcha: the caller must read the flag immediately after calling `advance()` or the completion signal is lost. A cleaner design has `advance()` return a `Bool` (or result enum) and eliminates the stored flag entirely.

**Magic numbers**
Phase values `0`–`5` and `-3` through `-1` are undocumented integers scattered through the codebase. Expressing them as a `CaseIterable` enum makes the valid states self-documenting and exhaustively checked by the compiler.

## `Audio.swift`

**Force unwrap of a resource path**
```swift
let filepath = Bundle.main.path(forResource: filename, ofType: nil)
let url: URL? = URL(fileURLWithPath: filepath!)  // crashes if asset is missing
```
`filepath` is `String?`. Force-unwrapping it crashes the app if the asset is absent. Use `guard let` and propagate the failure gracefully, or switch to the modern `Bundle.main.url(forResource:withExtension:)` which returns an optional `URL` directly and eliminates the intermediate string.

**Redundant `url != nil` check**
`URL(fileURLWithPath:)` is non-failable and never returns `nil`. The `if url != nil` branch is dead logic. After fixing the force unwrap above, this check can be removed entirely.

**Free function rather than a typed factory**
`makeAudioPlayer(filename:)` is a file-scoped free function. A `static func` on `AVAudioPlayer` (via extension) or a method on `SoundEffects` itself would be more discoverable and better encapsulated.

## `SoundEffects.swift`

**Dictionary of double-optionals**
```swift
var sounds = ["a4-note": makeAudioPlayer(...), "a3-note": makeAudioPlayer(...)]
// Inferred as [String: AVAudioPlayer?]
```
Subscripting a `[String: AVAudioPlayer?]` produces `AVAudioPlayer??`. The `if let sound = sounds[name]` call unwraps the outer optional, leaving `sound` as `AVAudioPlayer?`, which then requires optional chaining. Nils should be filtered out at construction so the dictionary is typed as `[String: AVAudioPlayer]`.

**Magic string keys**
`"a4-note"` and `"a3-note"` are repeated across `SoundEffects` and both `ContentModel` files. A small enum or static constants would make typos a compile error.

**Missing concurrency annotation**
`SoundEffects` is called from within a `Timer` closure running on the main run loop, but has no isolation annotation. This is a latent Swift 6 concurrency warning.

## `ContentModel.swift` (both targets)

**`@Observable` missing `@MainActor`**
Same rule as `Phase` — both `ContentModel` classes need `@MainActor`.

**Timer closure creates a retain cycle**
```swift
timer = Timer(timeInterval: 1, repeats: true) { _ in
    self.phase.advance()  // strong capture of self
    ...
}
```
`ContentModel` owns `timer`, and `timer` captures `ContentModel` strongly. `deinit` breaks the cycle by invalidating the timer, but `[weak self]` makes the relationship explicit and is safer if the ownership ever changes.

**`init()` starts the timer immediately**
`startCounting()` is called from `init()`. This means instantiating `ContentModel` in a preview, a test, or accidentally twice creates a live timer with real audio side-effects. The model should be inert at construction; counting should require an explicit call to `startCounting()`.

**Timer callback body is copy-pasted verbatim between targets**
Every audio cue and phase check is identical across both platform files. This is the most significant maintainability problem in the project. A shared `func tick()` method on a base class or in a protocol extension would eliminate the duplication.

**watchOS: `session` is an implicitly-unwrapped optional unnecessarily**
```swift
private var session: WKExtendedRuntimeSession!
```
`session` can be `WKExtendedRuntimeSession?` with optional chaining in `stopCounting()`.

**watchOS: inconsistent nil guard before force unwrap**
iOS uses `if timer != nil { RunLoop.main.add(timer!, ...) }` — a force unwrap inside the very guard meant to protect against it. watchOS uses `guard timer != nil else { return }` then still force-unwraps. Both should use `if let timer`.

**Spoken rep number is implicitly off-by-one**
```swift
self.speaker.speak(AVSpeechUtterance(string: String(self.count + 1)))
```
At the start of rep N's engage phase, `count` holds N−1 completed reps, so `count + 1` speaks the upcoming number. This is correct but non-obvious. A local `let nextRep = count + 1` with a brief comment would clarify the intent.

**iOS `ContentModel` inherits from `NSObject` without reason**
The watchOS version inherits `NSObject` to conform to `WKExtendedRuntimeSessionDelegate`. The iOS version has no such requirement and should not inherit from `NSObject`.

## `CoreContentView.swift`

**`Font.custom("SF Pro", size:)` bypasses Dynamic Type**
```swift
.font(.custom("SF Pro", size: imageFontSize))
.environment(\.font, Font.custom("SF Pro", size: fontSize))
```
Accessing the system font by name is unreliable — the font name varies by platform and may not resolve correctly. Use `.system(size:)`. More importantly, hard-coded point sizes opt the entire UI out of the user's Dynamic Type preference, which is an accessibility regression.

**Two different mechanisms for the two font sizes**
`imageFontSize` is passed as a constructor parameter; the body font is injected via environment. These two sizing mechanisms are inconsistent and make the callsites harder to reason about.

**`repCount` is owned by the wrong view**
`CoreContentView` holds `@State private var repCount`, a value that only `RepCounterView` can write — and only when the user taps Stop. The parent holds state that only the child can meaningfully update. The rep count (or the model that owns it) should live at the same level as, or above, the view that displays it.

---

## `RepCounterView.swift`

**Shared code has an implicit dependency on a platform-specific type**
```swift
@State var model = ContentModel()
```
`RepCounterView` lives in `Core/` (shared between targets), but `ContentModel` is defined separately in each platform target. This works because both targets happen to expose a type with the same name and interface, but it is fragile and invisible to the compiler. A protocol — e.g. `RepCounterModel` — defined in `Core/` and conformed to by both platform `ContentModel` types would make this contract explicit and enforced.

## App entry points (`TickerApp.swift`)

**Watch app struct name**
`Ticker_Watch_AppApp` — the Xcode-generated double-`App` suffix with an underscore was left in. `TickerWatchApp` would be cleaner.

**Audio session failure is silently swallowed**
A failure in `AVAudioSession` setup is only `print`ed. For a fitness app where audio is the entire feedback mechanism, this failure deserves more visible handling.

---

## Tests

All four test files are Xcode boilerplate with zero actual tests. `Phase` alone has straightforwardly testable behaviour: phase sequencing, wrap-around after phase 5, cycle completion signalling, and `label()` output for every state. These should be unit tested — they represent the core correctness guarantee of the app.

## Priority summary

### High — correctness or crashes
- Force unwrap in `Audio.swift` on a missing resource path
- Timer retain cycle in `ContentModel` (mitigated by `deinit`, but unsafe)
- `@Observable` classes missing `@MainActor` (Swift 6 concurrency violations): `Phase`, both `ContentModel`s

### Medium — maintainability
- Duplicated timer callback body across both `ContentModel` files
- `RepCounterView` implicitly depends on a platform-specific type; needs a protocol
- `CycleCompleted()` naming and flag-based cycle detection design
- Magic integer phase values (should be an enum)
- Magic string sound keys

### Lower — design quality
- `Font.custom("SF Pro")` bypassing Dynamic Type and accessibility
- `init()` starting the timer as a side-effect of construction
- `SoundEffects` dictionary of double-optionals
- `repCount` state owned by the wrong view
- No unit tests for `Phase`
